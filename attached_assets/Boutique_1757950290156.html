Project Documentation: Boutique Manager

Version: 2.0 (Hero Edition)

1. Introduction

1.1. Project Vision

To create a simple, powerful, and private desktop application that empowers a boutique owner to manage their business efficiently, replacing physical notebooks with a streamlined digital solution.

1.2. The "Zero to Hero" Path

  • The "Zero" (MVP): A reliable tool to store customer data and order history.
  • The "Hero" (Final Product): A complete business management suite with workflow tracking, visual references, profit calculation, and automated customer communication.

1.3. Core Value Proposition

"Boutique Manager" is an all-in-one digital assistant that helps you track orders, understand your finances, and provide a modern, professional experience to your customers, all from the privacy of your own computer.

2. Technology Stack

  • Backend Server: Java (with Spring Boot)
  • Frontend UI: React
  • Desktop App Wrapper: Electron.js
  • Database: SQLite
  • SMS/WhatsApp Integration: Twilio API

3. Features & Functionality

3.1. Core Functionality (The Foundation)

  • Customer Profiles: Securely save customer name, phone number, and detailed measurements. Each customer gets a unique, searchable ID.
  • Order History: Link multiple orders to each customer, recording the description, price, and date of each order.
  • Fast Search: Instantly find any customer by their name or phone number.

3.2. Enhanced Workflow Features

  • Order Status Tracking: Manage the entire lifecycle of an order with statuses like "New," "Cutting," "Stitching," "Ready for Pickup," and "Completed."
  • Image Uploads: Attach a photo to each order (e.g., a design reference, the fabric used, or the final product) for a perfect visual record.
  • Customer Notes: A dedicated section on each customer's profile to save personal preferences and important details.

3.3. Business Management Features

  • Expense Tracking: For each order, add an optional materialCost field to track expenses.
  • Profit Calculation: The "Monthly Report" will be upgraded to show Total Sales, Total Material Costs, and the final Net Profit.

3.4. Professional "Wow" Feature

  • Automated Customer Notifications: When an order's status is changed to "Ready for Pickup," the application will automatically send a professional SMS or WhatsApp message to the customer.

4. Database Schema Design (SQLite)

Customers Table

id (Primary Key, Auto-increment)
name (TEXT, NOT NULL)
phone (TEXT, UNIQUE)
measurements (TEXT)
notes (TEXT)

Orders Table

id (Primary Key, Auto-increment)
customer_id (INTEGER, Foreign Key to Customers.id)
description (TEXT, NOT NULL)
price (REAL)
material_cost (REAL)
order_date (DATE)
status (TEXT)
image_path (TEXT)

5. API Endpoint Specification (Java REST API)

  • GET /api/customers - Get all customers.
  • POST /api/customers - Add a new customer.
  • GET /api/customers/:id - Get a single customer's details.
  • GET /api/customers/:id/orders - Get all orders for a specific customer.
  • POST /api/customers/:id/orders - Add a new order (handles image upload).
  • PUT /api/orders/:id/status - Update the status of a single order.
  • GET /api/reports/monthly - Get the sales, costs, and profit.
  • POST /api/notifications/send - Send SMS/WhatsApp notification.

6. "Zero to Hero" Development Roadmap

  • Milestone 1 (Weeks 1-2): Backend Foundation (Core API for Customers & Orders).
  • Milestone 2 (Weeks 3-4): Core Frontend (React UI for MVP features).
  • Milestone 3 (Week 5): Add Workflow Features (Status Tracking, Image Uploads).
  • Milestone 4 (Week 6): Build the Business Dashboard (Profit Calculation).
  • Milestone 5 (Weeks 7-8): The "Wow" Feature & Final Packaging (Twilio API, Electron Builder).
